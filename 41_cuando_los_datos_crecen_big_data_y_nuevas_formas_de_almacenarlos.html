4.1 Cuando los datos crecen: Big Data y nuevas formas de almacenarlos

A lo largo del recurso has aprendido los fundamentos esenciales de cada fase del desarrollo de software, y ahora queremos detenernos en un aspecto muy relevante.

Durante el proceso también has aprendido a diseñar aplicaciones apoyadas en bases de datos relacionales, pero ¿sabías que no son la única opción? Existen otras formas de almacenar y gestionar grandes cantidades de datos que cada vez son más utilizadas, especialmente en entornos donde los datos crecen a gran velocidad o tienen estructuras muy diversas.

En este punto, conocerás brevemente algunas de estas alternativas mientras trabajas con la información que se te presenta sobre ellas. ¡Vamos a descubrir nuevas formas de organizar la información!

1. 🆚 Clásico vs Moderno: ¿SQL o NoSQL?

🎥 Equipo SQL vs Equipo NoSQL

¿Quieres entender las diferencias entre una base de datos relacional y una NoSQL? ¡Este vídeo lo explica con una dinámica muy divertida!

*Recuerda que puedes activar los subtítulos del vídeo si lo necesitas.

Tras ver el vídeo, observa ahora este cuadro comparativo con las características principales que diferencian las bases de datos SQL y NoSQL:

Característica SQL NoSQL
Almacenamiento de datos De manera estructurada Con su formato original
Capacidad de escalar Baja (centralizadas) Alta (distribuidas y pensadas para Big Data)
Adaptación a los cambios Poco y complejo Totalmente flexibles
Homogeneización Estandarizadas No son homogéneas
Aplicaciones De todo tipo Enfocadas al Big Data
Rapidez Menor Mayor
