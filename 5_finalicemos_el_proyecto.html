  2. Realizar la consulta a la base de datos:
    respuesta = self.cliente1.consulta(datos)
    Este método buscará el cliente cuyo DNI coincida y devolverá sus datos.
  3. Si la consulta devuelve datos (cliente encontrado):
    Debes rellenar todos los campos usando los valores devueltos. Los datos están en respuesta[0] y su orden corresponde con el orden de la tabla. Ejemplo:

    • self.telefono.set(respuesta[0][2]) → Teléfono
    • self.dni.set(respuesta[0][3]) → DNI
    • self.fecha.set(respuesta[0][4]) → Fecha de nacimiento
    • self.correo.set(respuesta[0][5]) → Correo electrónico
    • self.direccion.set(respuesta[0][6]) → Dirección
  4. Si no se encuentra ningún cliente con ese DNI:
    Deja vacíos todos los campos utilizando set('') y muestra un mensaje informativo:

    self.apellidos.set('')

Prueba y mantenimiento: dos fases clave para mejorar tu proyect

En el desarrollo de software, hacer pruebas no es opcional. Es la única forma de asegurarnos de que nuestra aplicación funciona correctamente y cumple lo que se espera de ella. Las pruebas ayudan a detectar errores, mejorar la calidad del programa y a ganar confianza tanto para ti como para quien usará la aplicación.

Una vez que el software pasa las pruebas y se entrega al cliente, entra en la fase de mantenimiento. A veces se piensa que el mantenimiento solo sirve para corregir errores, pero en realidad, también incluye mejoras, nuevas funciones y adaptaciones. La gran diferencia es que:

  • Las pruebas se hacen antes de entregar el software al cliente.
  • El mantenimiento se realiza después, cuando el programa ya está en uso.

En esta última etapa del proyecto aprenderás en qué consisten ambas fases, por qué son tan importantes y cómo afectan al ciclo de vida de cualquier aplicación.

Los tipos de pruebas

Vamos a conocer dos tipos de pruebas:

  • Pruebas de caja negra: Son las pruebas que habéis realizado en el punto 1. Sólo analizan las salidas de un sistema obtenidas a partir de ciertas entradas. Toman el código desarrollado como una caja negra en la que no importa el contenido.
  • Pruebas de caja blanca: Estas pruebas comprueban el código en sí mismo: verifican que se pasa por todas las posibles condiciones de if y while, los valores límite de los bucles for, etc. Estas pruebas exigen conocer perfectamente la estructura interna de las funciones que se están probando y requieren de mucho más tiempo que las de caja negra, por eso son menos habituales. En general, sólo se realizan cuando hay algún software crítico que pueda suponer grandes costes o vidas humanas.

Mantenimiento

Cuando se ha terminado y entregado un proyecto de software al cliente, generalmente necesita alguna modificación posterior. Esta fase es el mantenimiento.

Algunas veces, estas modificaciones se refieren a fallos que deben ser corregidos (denominados bugs); y otras veces se busca incorporar funcionalidades que les resultarían útiles a los usuarios. En estos casos se propone una ampliación del proyecto. Por esto, la fase de mantenimiento puede ser de cuatro tipos diferentes:

  • Correctivo: Diagnosticar y solucionar errores detectados por los usuarios.
  • Perfectivo: Implementar mejoras sugeridas por los clientes.
  • Adaptativo: Modificar el programa para que pueda ser usado en otro entorno.
  • Preventivo: Mejorar la fiabilidad para minimizar el riesgo de de aparición de posteriores problemas.
